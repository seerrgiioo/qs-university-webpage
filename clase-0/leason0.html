1.1 RegisterCommand

Introduction: This function creates a command that can be used by players in-game to trigger actions.


RegisterCommand('tpplayer', function(source, args)
    -- Code executed when the command is used
end)
  • source: ID of the player who executed the command.
  • args: An array containing additional arguments passed with the command.

1.2. Arrays and Tables

Purpose: Organizes multiple teleport locations for easier management and reference.


local teleportLocations = {

1.3 Conditional Statements

Purpose: Validates input parameters to prevent errors.


if targetId and locationIndex and teleportLocations[locationIndex] then
    -- Execute teleport logic
else
    -- Show error message
end
  • targetId: Ensures the player ID is valid.
  • locationIndex: Checks if the location index exists.
  • teleportLocations[locationIndex]: Validates the teleport location.

1.4 SetEntityCoords

Purpose: Moves the player to the specified coordinates.


SetEntityCoords(PlayerPedId(), location.x, location.y, location.z, false, false, false, true)
  • PlayerPedId(): Retrieves the current player entity.
  • location.x, location.y, location.z: Coordinates for the teleportation.
  • Flags Explained:
    • false: No rotation around the X-axis.
    • false: No rotation around the Y-axis.
    • false: No rotation around the Z-axis.
    • true: Clears objects around the destination.

1.5 TriggerClientEvent

Purpose: Sends an event from the server to a specific client.


TriggerClientEvent('teleportPlayer', targetId, location)
  • 'teleportPlayer': Name of the custom event.
  • targetId: Target player ID.
  • location: Data passed to the client (destination).
üí° Tip: Use descriptive event names to keep your code organized and understandable.

2.2 client.lua

Beginner Section: Simple Teleport Script

For beginners, this section focuses on the core teleportation functionality without advanced features like particle effects or validations.


-- Configuration for teleport locations
local teleportLocations = {
    { x = -75.0, y = -819.0, z = 326.0, name = "Main Rooftop" },
    { x = -428.0, y = 1111.0, z = 327.0, name = "Vinewood" },
    { x = 402.0, y = -996.0, z = -100.0, name = "Hidden Bunker" }
}

-- Register the /tp command
RegisterCommand('tp', function(source, args)
    local locationIndex = tonumber(args[1])
    if locationIndex and teleportLocations[locationIndex] then
        local location = teleportLocations[locationIndex]
        SetEntityCoords(PlayerPedId(), location.x, location.y, location.z, false, false, false, true)
        TriggerEvent('chat:addMessage', {
            color = { 0, 255, 0 },
            multiline = true,
            args = { "Teleport", "You've been teleported in: " .. location.name }
        })
        print("You've been teleported in: " .. location.name)
    else
        TriggerEvent('chat:addMessage', {
            color = { 255, 0, 0 },
            multiline = true,
            args = { "Error", "Position not valid! Use /tp [1-3]." }
        })
        print("Position not valid! Use /tp [1-3].")
    end
end)

Explanation:

  • RegisterCommand: Registers the /tp command.
  • teleportLocations: A list of teleport destinations with coordinates and names.
  • SetEntityCoords: Teleports the player to the specified location.
Tip: This simplified example is suitable for beginners. To add visual effects and advanced validation, see the advanced example below.

-- Configuration for teleport locations
local teleportLocations = {
    { x = -75.0, y = -819.0, z = 326.0, name = "Main Rooftop" },
    { x = -428.0, y = 1111.0, z = 327.0, name = "Vinewood" },

-- Register the /tp command
RegisterCommand('tp', function(source, args)
    local locationIndex = tonumber(args[1])
    if locationIndex and teleportLocations[locationIndex] then
        local location = teleportLocations[locationIndex]
        RequestNamedPtfxAsset("scr_rcbarry2")
        while not HasNamedPtfxAssetLoaded("scr_rcbarry2") do
            Citizen.Wait(0)
        end
        UseParticleFxAssetNextCall("scr_rcbarry2")
        StartParticleFxNonLoopedAtCoord(
            "scr_clown_appears",
            GetEntityCoords(PlayerPedId()),
            0.0, 0.0, 0.0,
            1.0, false, false, false
        )
        SetEntityCoords(PlayerPedId(), location.x, location.y, location.z, false, false, false, true)
        TriggerEvent('chat:addMessage', {
            color = { 0, 255, 0 },
            multiline = true,
            args = { "Teleport", "You've been teleported in: " .. location.name }
        })
        print("You've been teleported in: " .. location.name)
    else
        TriggerEvent('chat:addMessage', {
            color = { 255, 0, 0 },
            multiline = true,
            args = { "Error", "Position not valid! Use /tp [1-3]." }
        })
        print("Position not valid! Use /tp [1-3].")
    end
end)

Note: This script is compatible with Game Build 2372 or higher.

Explanation:

  • RegisterCommand: Registers the /tp command.
  • teleportLocations: A list of teleport destinations with coordinates and names.
  • SetEntityCoords: Teleports the player to the specified location.
  • StartParticleFxNonLoopedAtCoord: Creates a visual effect during the teleport.
Tip: Customize the teleportLocations array to add your own destinations.

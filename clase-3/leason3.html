A Step-by-Step Guide to Create a Simple Loading Screen in FiveM

This guide will show you how to create a basic loading screen for your FiveM server.
You'll learn how to create a loading screen with HTML, CSS, and JavaScript.
The loading screen will display a message and a progress bar.

1.1 Creating the Loading Screen

First, let's create the HTML structure for our loading screen:

<head>
    <link href="style.css" rel="stylesheet" type="text/css" />
</head>
<body>
    <div class="backdrop">
        <div class="main-container">
            <h2>Server Name</h2>
            <h3></h3>
            <div class="loadbar"><div class="loading-bar"></div></div>
        </div>
        <div class="bottom">
            <div id="gradient">
            </div>
        </div>
    </div>

    <script src="script.js"></script>
</body>

At this point, you should have a basic HTML structure for your loading screen. At head tag, we include the CSS file for styling. At body tag, we include the JavaScript file for the loading screen logic.

1.3 JavaScript Loading Screen

Now, let's create the JavaScript logic for our loading screen. Since this is before the server has started, we need add the script to the HTML.

var count = 0;
var thisCount = 0;

const emoji = {
    INIT_BEFORE_MAP_LOADED: [ 'üçâ' ],
    INIT_AFTER_MAP_LOADED: [ 'üçã', 'üçä' ],
    INIT_SESSION: [ 'üçê', 'üçÖ', 'üçÜ' ],
};

const handlers = {
    startInitFunctionOrder(data)
    {
        count = data.count;

        document.querySelector('.main-container h3').innerHTML += emoji[data.type][data.order - 1] || '';
    },

    initFunctionInvoking(data)
    {
        document.querySelector('.loading-bar').style.left = '0%';
        document.querySelector('.loading-bar').style.width = ((data.idx / count) * 100) + '%';
    },

    startDataFileEntries(data)
    {
        count = data.count;

        document.querySelector('.main-container h3').innerHTML += "\u{1f358}";
    },

    performMapLoadFunction(data)
    {
        ++thisCount;

        document.querySelector('.loading-bar').style.left = '0%';
        document.querySelector('.loading-bar').style.width = ((thisCount / count) * 100) + '%';
    },

};

window.addEventListener('message', function(e)
{
    (handlers[e.data.eventName] || function() {})(e.data);
});

We set the JavaScript logic for the loading screen. This includes handling events from the FiveM server and updating the loading screen elements accordingly.

FiveM has an entire docs page for this topic here. There you can see the documentation for the loading screen.

It includes the following events:

  • Cursor/input - You define this event in the fxmanifest.lua so the cursor can be enabled.

Handover data

How it works: When a player connects, the playerConnecting event is triggered on the server. Within this event, the deferrals.handover function is used to send data to the client in the form of key-value pairs. For instance, the server can send the player's name using the GetPlayerName(source) function. This data is passed to the client seamlessly. Here you have an example:

! THIS IS ONLY A LITTLE EXPLANATION !

At Client-Lua:

AddEventHandler('playerConnecting', function(_, _, deferrals)
    local source = source
    deferrals.handover({
        name = GetPlayerName(source)
    })
end)

At js:

<!-- loading screen page -->
<h1 id="namePlaceholder">Welcome, <span></span></h1>

<script type="text/javascript">
window.addEventListener('DOMContentLoaded', () => {
    console.log(`You are connecting to ${window.nuiHandoverData.serverAddress}`);

    // Use innerText to safely display user-provided data
    document.querySelector('#namePlaceholder > span').innerText = window.nuiHandoverData.name;
});
</script>

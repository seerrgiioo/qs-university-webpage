1. Introduction

Tables in Lua are a flexible and essential data structure. They can act as arrays, dictionaries, or even both simultaneously. Lua tables are untyped, dynamic, and indexed, making them versatile for various programming tasks.

2. Creating Tables

Tables are created using curly braces {}. Here’s how:


    -- Create an empty table
    local myTable = {}
    
    -- Create a table with values
    local fruits = {"apple", "banana", "cherry"}

Explanation:

  • Empty tables: {} creates a blank table. You can dynamically add key-value pairs or indexed values.
  • Indexed tables: Tables like {"apple", "banana", "cherry"} automatically assign numerical indices starting from 1.

3. Accessing Tables

Accessing table values depends on whether you're using indices (arrays) or keys (dictionaries).


    local fruits = {"apple", "banana", "cherry"}
    
    -- Access by index
    print(fruits[1]) -- Output: apple
    
    -- Using keys
    local person = {name = "John", age = 30}
    print(person.name) -- Output: John
    print(person["age"]) -- Output: 30

Explanation:

  • Index-based access: Use numerical indices (e.g., fruits[1]) to retrieve values from an indexed table.
  • Key-based access: Use dot notation (person.name) or bracket notation (person["name"]) for dictionaries.

4. Modifying Tables

You can add, update, or delete values in a table dynamically.


    local fruits = {"apple", "banana"}
    
    -- Adding a new element
    fruits[3] = "cherry"
    
    -- Updating an existing element
    fruits[1] = "grape"
    
    -- Deleting an element
    fruits[2] = nil -- Removes "banana"
    
    -- Printing updated table
    for i, fruit in ipairs(fruits) do
        print(i, fruit)
    end

Explanation:

  • Adding values: Specify an unused index or key to add new elements.
  • Updating values: Use an existing index or key to modify a value.
  • Removing values: Set the value to nil to remove it from the table.

5. Using the `table` Library

The table library provides built-in functions to manipulate tables. Below are the most commonly used functions:

5.1 table.insert

Adds an element to a table at a specified position.


    local fruits = {"apple", "banana"}
    
    -- Insert "cherry" at position 2
    table.insert(fruits, 2, "cherry")

How it works: Inserts "cherry" at index 2, shifting subsequent elements to the right.

5.2 table.remove

Removes an element from a table at a specified position.


    local fruits = {"apple", "banana", "cherry"}
    
    -- Remove the second element
    table.remove(fruits, 2)

How it works: Removes the element at index 2 ("banana"), and subsequent elements are shifted left.

5.3 table.sort

Sorts table elements in ascending order.


    local numbers = {3, 1, 4, 1, 5, 9}
    
    -- Sort the numbers
    table.sort(numbers)

How it works: Reorders elements in ascending order. For custom sorting, provide a comparator function.

5.4 table.concat

Concatenates table elements into a string.


    local fruits = {"apple", "banana", "cherry"}
    
    -- Join elements with a comma
    local result = table.concat(fruits, ", ")
    print(result) -- Output: apple, banana, cherry

How it works: Joins all elements with the specified separator (", ").

6. Looping Through Tables

There are two main methods to loop through tables: ipairs and pairs.

6.1 ipairs

Used for sequential tables (arrays). Iterates over numeric indices in ascending order.


    local fruits = {"apple", "banana", "cherry"}
    
    for i, fruit in ipairs(fruits) do
        print(i, fruit)
    end

How it works: ipairs stops when it encounters a nil value, ensuring sequential iteration.

6.2 pairs

Used for key-value tables (dictionaries). Iterates over all key-value pairs.


    local person = {name = "John", age = 30}
    
    for key, value in pairs(person) do
        print(key, value)
    end

How it works: pairs iterates over every key-value pair, regardless of order.

7. Conclusion

Tables in Lua offer unmatched flexibility. Whether you're building arrays, dictionaries, or hybrid structures, mastering tables is essential for any Lua developer. The table library and looping constructs further enhance table manipulation.
